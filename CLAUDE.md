# CLAUDE.md - 自媒体选题评价助手

## 项目概述
这是一个单文件 HTML 网页应用，帮助自媒体创作者从多维度评价选题价值，通过加权平均得出综合分数。

## 标准文件路径

| 文件 | 路径 | 用途 |
|------|------|------|
| 需求规格 | docs/requirements.md | 功能需求、评价维度定义、评分标准 |
| 技术规格 | docs/tech-spec.md | localStorage 结构、计算公式、数据模型 |
| 设计规范 | docs/design-spec.md | 色彩、排版、组件样式标准 |
| 执行计划 | docs/execution-plan.md | 分步开发任务清单与当前进度 |

## 开发日志
- 路径：`dev-logs/YYYY-MM-DD.md`
- 模板：日期、今日完成、遇到的问题、明日待办
- 每次开发前读取当天日志；每次开发后更新

## 产品文件
- `index.html` — 所有 HTML/CSS/JS 集中在一个文件（GitHub Pages 入口）

## 工作流程
1. **开发前**：读取 `docs/execution-plan.md` 确认当前步骤
2. **开发中**：参照 `docs/` 下相应标准文件（需求→tech-spec→design-spec）
3. **开发后**：更新 `dev-logs/YYYY-MM-DD.md` 当日日志
4. **步骤完成**：更新 `docs/execution-plan.md` 中的步骤状态

## 核心原则
- **小步推进**：每次只完成一个可验证的步骤，验证通过后再进入下一步
- **先标准后代码**：所有开发行为必须有 docs/ 中的标准作为依据
- **每日记录**：每次改动后更新开发日志
- **用户友好**：用户是编程小白，代码注释要清晰，功能要直观
